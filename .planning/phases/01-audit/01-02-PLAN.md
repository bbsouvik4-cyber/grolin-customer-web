**Phase A: Identify all broken elements from code + known issues**

Using the AUDIT-INVENTORY.md from Plan 01 and the CONCERNS.md known issues list, compile every broken element. For EACH of the following known issues, read the specific source file to confirm the issue and extract exact file path + line number + code snippet:

1. **Hero image 404s**: Read `src/components/home/HeroLayered.tsx`. Find every `<Image src=` or `src=` attribute. Document the exact URL/path used. Check if the referenced image files exist in `public/`. If they reference external URLs (like `grocery-api.shotlin.in`), document as potential 404/CORS. Record exact line numbers.

2. **CORS on category images**: Read `src/services/categories.service.ts` and any component that renders category images (e.g., `CategoryRow.tsx`, `HomeCategoryGrid.tsx`). Find where image URLs are constructed. If they use `grocery-api.shotlin.in` domain, document the exact URL pattern and line number.

3. **Category page infinite spinner**: Read `src/app/(shop)/categories/[id]/page.tsx` and `src/app/(shop)/categories/[id]/loading.tsx`. Document what triggers the loading state and what condition might cause it to never resolve (e.g., CORS blocking the API response).

4. **Blank product detail**: Read `src/app/(shop)/products/[slug]/page.tsx`. Document what happens when the product is not found — is there a fallback UI or does it render blank? Record exact line numbers where the blank state occurs.

5. **Empty product grid**: Read `src/app/(shop)/products/page.tsx` and `src/components/product/ProductGrid.tsx`. Document the empty state handling — does it show a beautiful empty state or a white void?

6. **Collection images**: Check if `public/images/collections/` directory exists and contains `col-*.webp` files. Read `src/components/home/CollectionRow.tsx` to find image references. Document any missing files.

7. **Missing loading.tsx files**: Run `find src/app -name "loading.tsx"` to confirm only `categories/[id]/loading.tsx` exists. List every route that lacks a loading.tsx.

8. **LazyMotion strict mode violations**: From the AUDIT-INVENTORY.md motion import audit (section 5.3), compile every file that imports `motion` instead of `m` from framer-motion. These are strict mode violations.

Classify slop patterns into these categories (from UI-SPEC section 3):
- **Flat white sections**: `bg-white` used as page background (not card surface)
- **Generic grays**: `bg-gray-*`, `bg-neutral-*`, `bg-slate-*` on non-text elements

- **Gray skeletons**: `bg-gray-200` in skeleton components instead of warm tokens
- **Unsaturated neutrals**: Hardcoded `#F5F5F5`, `#EEEEEE`, etc. without warm tint
- **Accent misuse**: Purple or green used outside their reserved contexts
